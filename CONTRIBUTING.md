# Contributing

Thank you for your interest in contributing to Banany Blog.

This project is free and open for use, modification, and redistribution, but
all copies and derivative works must keep the original copyright and license
notice so credit remains with the original author and contributors.

## Ground Rules

- Be respectful and constructive.
- Keep changes focused and easy to review.
- Do not remove or alter existing copyright or license notices.
- If you add code from another source, make sure you have the right to use it
  and document the source clearly.

## Development Setup

1. Clone the repository.
2. Install dependencies:

```bash
npm install
```

3. Configure the required environment variables.
4. Start the development server if needed:

```bash
npm run dev
```

## Before You Submit

Please make sure your changes are working and do not break the project.

Run:

```bash
npm test
npm run lint
npm run build
```

## Contribution Workflow

1. Fork the repository or create a branch from the main development branch.
2. Make your changes in small, clear commits.
3. Write clear commit messages in English.
4. Add or update tests when behavior changes.
5. Open a pull request with:
   - a short summary of the change
   - the reason for the change
   - testing notes
   - screenshots if the UI changed

## Code Style

- Follow the existing project structure and naming conventions.
- Prefer simple, readable solutions over clever ones.
- Keep user-facing text and error messages clear.
- Avoid unrelated refactors in the same pull request.

## Questions

If you want to discuss a feature or a large change before implementing it,
open an issue or start a discussion first.
